# PRD — VOC 기반 AI Assistant

## 1. 어떤 서비스인가요 (한 줄)
발리에서 서핑 레슨을 운영하는 Good Day Surf Bali에 접수된 고객의 소리(VOC) 원문을 입력하면 AI가 감정·요약·키워드·카테고리·긴급도를 자동 분석해 카드로 보여주고, 쌓인 VOC를 대시보드에서 한눈에 조회할 수 있는 서비스입니다.

## 2. 작성자
- 이름: 한지아
- 작성일: 2026-07-15

## 3. 사용자
- 1차 사용자: 본인 (Good Day Surf Bali 운영자 역할 가정)
- 사용 장면: 리뷰·카톡·이메일 등으로 접수된 VOC 원문을 붙여넣고, 감정·긴급도·카테고리를 즉시 파악해 우선순위를 정할 때

## 4. 핵심 기능 (우선순위)
- P0. VOC 입력 폼: 원문 텍스트를 붙여넣고 분석 요청
- P0. AI 분석 (Gemini API): 감정(긍정/중립/부정), 한 줄 요약, 키워드 3~5개, 카테고리(강습/강사·예약/일정·안전/장비·가격/환불·사진/영상·기타), 긴급도(상/중/하)
- P0. 분석 결과 카드: 입력 직후 결과를 카드 형태로 표시
- P0. Supabase 저장: 원문 + 분석 결과를 `voc_entries` 테이블에 저장
- P1. 실시간 대시보드: 저장된 VOC 목록을 카드/테이블로 조회, 카테고리·긴급도별 필터
- P1. 긴급도 "상" 하이라이트: 목록 상단에 별도 강조 표시

### 카테고리 정의
- **강습/강사**: 레슨 내용, 강사 태도/실력
- **예약/일정**: 예약 프로세스, 시간 변경, 노쇼
- **안전/장비**: 안전관리, 웨트슈트·보드 등 장비 상태
- **가격/환불**: 요금, 환불 분쟁
- **사진/영상**: 세션 촬영 품질·전달 (`Every wave, remembered` 핵심 가치와 직결)
- **기타**: 위에 해당하지 않는 VOC

### 분석 결과 스키마 (예시)
```
{
  "sentiment": "긍정 | 중립 | 부정",
  "summary": "한 줄 요약",
  "keywords": ["키워드1", "키워드2", "키워드3"],
  "category": "강습/강사 | 예약/일정 | 안전/장비 | 가격/환불 | 사진/영상 | 기타",
  "urgency": "상 | 중 | 하"
}
```

## 5. 기술 스택
- 프론트엔드: Next.js 15 + TypeScript + Tailwind CSS
- 백엔드: Next.js API Routes (`/api/voc/analyze`)
- AI 분석: Gemini API (Google AI Studio, 무료 티어) — 키는 `.env.local`의 `GEMINI_API_KEY`, 코드에 하드코딩 금지
- 데이터 저장: Supabase (`voc_entries` 테이블), `good-day-surf-brand`와 동일한 프로젝트/키 정책 — anon/publishable key로 클라이언트에서 조회, 쓰기는 서버 API Route 경유

## 6. 화면 구성 (단일 페이지, `/`)
- 상단: 타이틀("VOC AI Assistant") · 부제
- VOC 입력 영역: 텍스트 영역 + "분석하기" 버튼
- 분석 직후: 방금 분석한 결과를 카드로 즉시 표시 (감정 배지, 요약, 키워드 태그, 카테고리, 긴급도 배지)
- 하단: 대시보드 — 누적 VOC 리스트 (최신순), 카테고리/긴급도 필터, 긴급도 "상" 항목 상단 하이라이트
- 디자인 규칙: `surface-*`/`text-*`/`accent-*`/`border-*` 토큰 구조를 쓰되, **색상 값은 `good-day-surf-brand/design.md`의 브랜드 팔레트(크림·코코아·틸·오렌지)를 기본값으로 삼는다** (`app/globals.css`, 이 프로젝트는 Good Day Surf Bali 전용이라 별도 스코프 없이 `:root`가 곧 브랜드 팔레트)
  - 배경(크림 #FDF8EC) · 카드(크림 #FAF0DC) · 텍스트(코코아 #452A1B/#6B4630) · CTA·강조(오렌지 #DE672B) · hover·포커스(틸 #1B5449/#2E7C6E)
  - 다크 모드 값은 design.md에 정의가 없어 브랜드 4색(크림/코코아/틸/오렌지) 안에서 명암을 뒤집어 자체 도출 (`app/globals.css`의 `[data-theme="dark"]` 참고)
  - 하드코딩 hex 금지 원칙 유지 — 반드시 CSS 변수(토큰)를 거쳐서만 사용
  - 한국어 / 카드형·심플 / 업무 톤 / 모바일 반응형
  - 모서리는 각진 사각형(rounded-none) 통일 — 로딩 스피너만 예외 (design.md의 둥근 radius 시스템은 색상 이관 범위 밖)

## 7. 이 정도면 합격 (완성 기준)
- VOC 원문을 입력하고 "분석하기"를 누르면 5초 안에 분석 결과 카드가 보인다
- 분석 결과가 Supabase에 저장되고, 새로고침해도 대시보드 목록에 남아있다
- 에러 시(분석 실패, 저장 실패) 사용자에게 안내 문구가 보인다

## 8. 서비스가 지켜야 할 선
- VOC 원문에 포함될 수 있는 개인정보(이름·연락처 등)는 화면에 그대로 노출하지 않도록 추후 마스킹 검토 (1차 범위는 원문 그대로 저장하되, 접근은 본인만 가능한 전제)
- API 키(Gemini, Supabase service role 등)는 서버에서만 사용하고 화면·코드에 노출하지 않는다

## 9. 1주일 뒤 달라지는 모습
접수되는 VOC를 수작업으로 읽고 분류하지 않아도, 입력 즉시 감정·긴급도·카테고리가 정리되어 우선순위가 높은 건을 먼저 확인하게 된다.

## 10. 구현 밖 범위 (Out of Scope)
- 로그인/권한 분리, VOC에 대한 자동 답변 생성, 실제 고객 알림 발송은 오늘 범위 아님
- VOC 원문 자동 수집(이메일/게시판 연동)은 범위 밖 — 수동 입력만 지원

## 11. 구현 메모 (시간순)
- **AI 분석 엔진 변경**: 최초 §5에는 Claude API(Anthropic)로 적혀 있었으나, API 키 발급 비용 때문에 무료 티어가 있는 Gemini API(Google AI Studio)로 변경. `.env.local`의 `GEMINI_API_KEY` 사용 (2026-07-15).
- **카테고리 재설계**: 최초 구현 시 카테고리를 쇼핑몰 기준(제품/배송/CS응대/가격/기타)으로 잘못 설계했다가, 실제 서비스가 발리 서핑 레슨(Good Day Surf Bali)임을 반영해 강습/강사·예약/일정·안전/장비·가격/환불·사진/영상·기타로 교체. `lib/voc/types.ts`, `lib/voc/geminiClient.ts` 프롬프트, Supabase `voc_entries` 테이블의 `category` CHECK 제약까지 함께 갱신 (2026-07-15).
- **Gemini 모델 교체**: 응답 시간이 7~30초로 들쭉날쭉하고 간헐적으로 502가 발생하던 문제(§7 "5초 이내" 기준 미달)의 원인은 `gemini-flash-latest`(gemini-3.5-flash)의 무료 티어 **일일 요청 한도(20건/일)** 초과였다. `gemini-flash-lite-latest`로 모델을 교체해 해결 — 응답 시간이 2~3초대로 개선됨 (2026-07-15).
- **컬러 팔레트 변경**: §6에 반영 — 범용 색상 대신 `good-day-surf-brand/design.md`의 실제 브랜드 팔레트(크림/코코아/틸/오렌지)를 적용 (2026-07-15).
- **독립 프로젝트로 분리**: 원래 `Next-Project`(News 등 다른 데이 프로젝트와 한 Next.js 앱을 공유) 안의 `/voc-assistant` 라우트로 시작했으나, Good Day Surf Bali 전용 서비스로 별도 배포·관리하기 위해 루트 레벨 독립 프로젝트(`VOC-AI-Assistant/`)로 이전. 라우트가 `/voc-assistant` → `/`(루트)로 바뀌었고, 공유 컴포넌트(`components/news/ui/*`)는 `components/ui/*`로 복제, 색상 토큰은 스코프 클래스(`.voc-theme`) 없이 `:root` 기본값으로 단순화됨. 아직 git 저장소·Vercel 배포는 안 된 상태 (2026-07-15).
